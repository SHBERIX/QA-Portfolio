# Отсутствует защита от CSRF при удалении заметки

## Описание

Обнаружена потенциальная уязвимость типа **Cross-Site Request Forgery (CSRF)**. Запрос на удаление заметки может быть выполнен без проверки CSRF-токена, что позволяет злоумышленнику инициировать удаление заметки от имени авторизованного пользователя.

---

## Шаги воспроизведения

1. Авторизоваться в веб-приложении.
2. Создать новую заметку.
3. Перехватить запрос на удаление заметки с помощью прокси (например, Burp Suite).
4. Убедиться, что запрос не содержит CSRF-токен.
5. Создать HTML-страницу, автоматически отправляющую аналогичный запрос:

```html
<form action="https://example.com/api/notes/delete/1" method="POST">
    <input type="submit" value="Delete">
</form>

<script>
document.forms[0].submit();
</script>
```

6. Открыть подготовленную страницу в браузере, оставаясь авторизованным в приложении.

---

## Фактический результат

Запрос выполняется успешно, а заметка удаляется без дополнительного подтверждения личности пользователя или проверки CSRF-токена.

---

## Ожидаемый результат

Приложение должно защищать все операции, изменяющие данные пользователя, от CSRF-атак.

Ожидаемое поведение:

- Для каждого изменяющего запроса должен использоваться уникальный CSRF-токен.
- Сервер должен отклонять запросы без корректного CSRF-токена.
- Дополнительно рекомендуется использовать атрибут cookie `SameSite` и проверку заголовков `Origin` или `Referer`.
